## Goal
A PR gate so every task lands green: formatting, analysis, and tests run on every push
and pull request. Do this immediately after 01 so all later PRs are gated.

## Scope
- in:
  - `.github/workflows/ci.yml`: checkout, Flutter setup (pin the version), `flutter pub
    get`, `dart format --set-exit-if-changed .`, `flutter analyze`, `flutter test`.
  - Cache pub dependencies for speed.
- out:
  - No build/release jobs, no device/integration tests yet.

## Acceptance criteria
- [ ] CI runs on push + PR and fails on format drift, analyzer issues, or test failures.
- [ ] A trivial PR shows the green check.
